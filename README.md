# Tg Bot CGI-like Server

Этот проект реализует сервер для Telegram-бота, работающего по принципу CGI: на каждое сообщение пользователя запускается отдельный процесс с скриптом bot.py, который формирует ответ. Бот представляет собой онлайн-магазин с возможностью просмотра товаров и оформления заказа.

## Предварительные требования

1. Зарегистрируйте бота в телеграм с помощью [@BotFather](https://t.me/BotFather)
2. Убедитесь, что на ПК установлен Python. Сделать это можно выполнив команду в терминале
    ```shell
    python --version
    ```
    Если видите ошибку вместо номера версии, попробуйте команду     
    ```shell
    python3 --version
    ```
    Если сработала вторая команда -- используйте её запуска кода в дальнейшем 
3. Если вы используете Windows и Python не установлен -- скачайте установочный файл 
[по ссылке](https://www.python.org/downloads/) и установите Python

## Инструкция по установке и запуску

### 1 Настройка токена

В файле `config_example.ini` замените текст `1111111111:YOUR-TgBotToken` на ваш токен, полученный в [@BotFather](https://t.me/BotFather) и переименуйте файл в `config.ini`.

### 2. Подготовка файлов товаров

- Все изображения товаров поместите в папку `products/`.
- Для каждого товара создайте текстовый файл с описанием (например, `cup_1.txt`) в той же папке.
- Убедитесь, что имена файлов совпадают с указанными в словаре `ITEMS` в файле `bot.py`.

### 3. Запуск бота

Находясь в папке проекта выполните команду

```bash
python server.py
```
Замените `python` на `python3`, если Python уже установлен, но не запускается из командной строки. В первую очередь эта
особенность применима для Linux и MacOS

## Структура проекта

- `server.py` – основной сервер, получающий обновления от Telegram и запускающий bot.py.
- `bot.py` – скрипт, который выполняется на каждое сообщение. В нём описана логика ответов.
- `config.ini` – файл конфигурации с токеном бота, прописан в `.gitignore`.
- `products/` – папка с изображениями и текстовыми описаниями товаров.

## Как работает бот (логика в `bot.py`)

Бот получает сообщение пользователя через `sys.argv[1]` и в зависимости от его содержимого формирует вывод.

## Команды

- **/start** – приветственное сообщение и кнопка перехода к списку товаров.
- **/list** – список всех доступных товаров в виде инлайн-кнопок (каждая кнопка ведёт на соответствующий товар).
- **Нажатие на кнопку товара** – отправляет изображение товара, его описание (из текстового файла) и кнопки "Заказать" и "Список товаров".
- **/order** – подтверждение заказа с сообщением, что с пользователем свяжется менеджер.
- **Любое другое сообщение** – считается обращением к менеджеру; бот отвечает стандартной фразой и предлагает вернуться к списку товаров.

## Формат вывода

Скрипт `bot.py` выводит строки, которые интерпретируются сервером:
- `Картинка: <путь_к_файлу>` – отправляет изображение.
- `Кнопка: <текст> --> <callback_data>` – создаёт инлайн-кнопку.
- Любой другой текст отправляется как обычное сообщение (или как подпись к последней картинке, если она была).

## Как добавить новый товар

1. Добавьте элемент в словарь `ITEMS` в `bot.py`:

```bash
"5": {
    "pk": 5,
    "title": "Название товара",
    "image": "products/имя_файла.jpg",
    "description": "products/имя_файла.txt",
}
```

2. Положите изображение `имя_файла.jpg` в папку `products/`.
3. Создайте файл `имя_файла.txt` с описанием товара в той же папке.
4. Перезапускать сервер не нужно – изменения применятся при следующем сообщении.

## Примечания

- Кодировка текстовых файлов описания – UTF-8.
- Все пути к файлам указываются относительно папки проекта.
- Бот автоматически удаляет предыдущие инлайн-кнопки при отправке нового сообщения с кнопками (чтобы интерфейс оставался чистым).
